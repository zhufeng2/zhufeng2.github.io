需要注意的地方

首先我们知道在使用爬虫的时候必须遵守其合法性，例如，别人的个人隐私信息，还有网站规定了不能爬取的内容,例如在淘宝中：

robots.txt表明了哪些信息能够爬取，哪些信息是不能够爬取的。

但是自己还是要衡量一下，避免违反法律。

爬虫实战分析

简易的网页采集器

首先，我们是对搜狗网页的html源码进行了爬取，这个实例较简单，我们可以通过这个分析爬虫工作的原理。

import requests #python中urllib包已经被涵盖在requests内，因此不需单独导入
url = "https://www.sogou.com/" # 向url中传入字符串，以便于后面的请求
rsepond = requests.get(url) # 这里我们处理的是get请求，这个看具体的网站而定，有时是post请求
print(respond.text)

如图所示的是一个get请求：

但是我们的网页采集器还是需要进行完善的。
第一个完善的是请求头需改成浏览器的请求头，这里就可以伪装成浏览器请求网站，是最简单的一种反反爬虫的策略。

'user-Agent' = "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/88.0.4324.190 Safari/537.36"

第二个改进就是根据用户输入的关键字来进行页面抓取：

我们发现关键字的抓取是根据关键字query来获得页面的。

get(url: Union[str, bytes], params: Union[Mapping....)

requests.get()中的关键字params就指定了这些特定的抓取内容，并且该关键字传入的是一个字典类型的数据。
最后一个优化就是采集的数据我们是需要存储的，利用python的文件操作我们可以实现这一点。
最后实现源码改进：

import requests #python中urllib包已经被涵盖在requests内，因此不需单独导入
headers = {
    'user-Agent' : "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, 					like Gecko) Chrome/88.0.4324.190 Safari/537.36"}
url = "https://www.sogou.com/" # 向url中传入字符串，以便于后面的请求
kw = input("please input a key words:")
param = {'query': kw}
respond = requests.get(url, params = param, headers = headers ) # 这里我们处理的是get请求，这个看具体的网站而定，有时是post请求
filename = kw +'.html'
with open(filename, 'w', encoding = 'utf-8') as fp:
    fp.write(respond.text)

百度翻译请求

我们注意到浏览器只是部分元素发生了改变，所以这里可能是发起了一个请求，然后服务器按照你发出的请求返回了一串数据.

我们对浏览器进行查看是一个post请求，并且我们进一步查看：

发现content-type返回了一串json数据，这串json数据被解析后应该就是翻译结果。目前发现百度翻译的请求网站已经看不到了，所以这里直接给出请求网站，我们需要做的就是爬取，解析，存储json数据。
实现代码如下所示:

import requests
import json
 headers ={ 'user-Agent' : "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 							(KHTML,like Gecko) Chrome/88.0.4324.190 Safari/537.36"}
 url = 'https://fanyi.baidu.com/sug'
# 我们分析网页可以得到是一个data类型
word = input('please input a word:')
data = {'kw':word}
respond = requests.post(url = url, data = data, headers = headers)
dic_obj = respond.json()
print(dic_obj)
